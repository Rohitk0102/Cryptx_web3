CryptX bootstrap requirements
=============================

Note:
- This repository is a Node.js monorepo. There are no Python or pip dependencies.
- This file is a machine-setup checklist for running the project on a new device.

Minimum software
----------------

Required:
- Node.js 18+ (Node.js 20 LTS recommended)
- npm 9+

Recommended for true one-click local setup on Windows:
- Docker Desktop with `docker compose`

Optional if you do not use Docker Desktop:
- PostgreSQL 15+ running on `localhost:5432`
- Redis 7+ running on `localhost:6379`

External keys you still need to provide
---------------------------------------

The startup script can create env files automatically, but these values are still project-specific:
- `apps/web/.env.local`
  - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
  - `CLERK_SECRET_KEY`
  - `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID`
- `apps/api/.env`
  - `CLERK_SECRET_KEY`

Optional feature keys:
- CoinDCX API keys
- OpenAI / Anthropic / Hugging Face keys
- CoinMarketCap / Covalent / Moralis / Gate.io keys

What `start.bat` does now
-------------------------

When you double-click `start.bat` on Windows, it will:
1. Check Node.js and npm
2. Install npm workspace dependencies if `node_modules` is missing
3. Create `apps/api/.env` from `apps/api/.env.example` if missing
4. Create `apps/web/.env.local` from `apps/web/.env.example` if missing
5. Repair `DATABASE_URL` and `REDIS_URL` in `apps/api/.env` to the expected local values
6. Start PostgreSQL and Redis with Docker Desktop when available
7. Run `npx prisma generate`
8. Run `npx prisma migrate deploy`
9. Fall back to `npx prisma db push` for local bootstrap if deploy fails
10. Start both backend and frontend with `npm run dev`

First run on a new Windows device
---------------------------------

1. Install Node.js.
2. Install Docker Desktop and make sure it is running.
3. Clone the repo.
4. Double-click `start.bat`.
5. If this is the first machine setup, fill in the generated env files:
   - `apps/api/.env`
   - `apps/web/.env.local`
6. Double-click `start.bat` again.

Useful commands
---------------

- `start.bat`
  - Full bootstrap plus app startup
- `start.bat setup`
  - Bootstrap only, without starting the dev servers
- `start.bat check`
  - Create missing env files if needed, then validate and repair local PostgreSQL and Redis endpoints
- `start.bat help`
  - Usage help

Local URLs
----------

- Frontend: `http://localhost:3000`
- Backend: `http://localhost:5000`
- Backend health check: `http://localhost:5000/health`

Notes
-----

- The backend `.env.example` now contains safe local-development placeholder secrets so Prisma and the API can boot on a fresh machine.
- Docker is the simplest path for a new device because it removes the need to install PostgreSQL and Redis manually.
- Authentication screens still require real Clerk keys from your own project.
